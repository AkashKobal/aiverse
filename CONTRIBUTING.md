# Contributing to AIVerse

Thank you for your interest in contributing to AIVerse. Contributions help improve the platform and make it more useful for everyone.

---

## How to Contribute

You can contribute in several ways:

* Reporting bugs
* Suggesting new features
* Improving documentation
* Fixing bugs
* Adding new AI model integrations
* Improving performance or UI

---

## Getting Started

### 1. Fork the repository

Click the **Fork** button on GitHub.

---

### 2. Clone your fork

```bash
git clone https://github.com/YOUR_USERNAME/aiverse.git
cd aiverse
```

---

### 3. Create a new branch

```bash
git checkout -b feature/your-feature-name
```

Examples:

```bash
feature/openai-integration
feature/ui-improvement
fix/controller-bug
```

---

### 4. Run the project locally

```bash
mvn spring-boot:run
```

Open:

```
http://localhost:6768
```

---

### 5. Make your changes

Follow these guidelines:

* Write clean and readable code
* Follow Java naming conventions
* Do not commit secrets or API keys
* Test your changes before committing

---

### 6. Commit your changes

Use clear commit messages:

```bash
git commit -m "feat: add OpenAI integration"
```

Commit message examples:

```
feat: add new model integration
fix: resolve controller issue
docs: update README
refactor: improve service structure
test: add unit tests
```

---

### 7. Push your branch

```bash
git push origin feature/your-feature-name
```

---

### 8. Create Pull Request

Go to GitHub and click **New Pull Request**

Provide:

* Description of changes
* Screenshots (if UI changes)
* Related issue number (if applicable)

---

## Contribution Guidelines

Please ensure:

* Project builds successfully
* No compilation errors
* No sensitive data committed
* Code follows project structure
* Changes are tested

---

## Reporting Bugs

Create a GitHub Issue and include:

* Description
* Steps to reproduce
* Expected behavior
* Screenshots (if applicable)

---

## Suggesting Features

Create a GitHub Issue with:

* Feature description
* Use case
* Expected benefit

---

## Code Style

Follow standard Java and Spring Boot conventions.

Keep code:

* Clean
* Readable
* Modular

---

## Questions

If you have questions, open a GitHub Issue.

---

Thank you for contributing to AIVerse.
